# Git 提交信息规范

当用户请求生成 Git 提交信息或进行代码提交时,请遵循以下规范:

## 语言要求

- 所有 Git 提交信息必须使用**简体中文**
- 提交信息要简洁明了,直接说明改动内容
- 避免使用英文,除非是专有名词或代码相关术语

## 提交信息格式

使用以下格式:

```
<类型>: <简短描述>

<详细说明>(可选)
```

## 提交类型(中文)

- `新增`: 新功能或新文件
- `修复`: Bug 修复
- `优化`: 代码优化、性能提升
- `重构`: 代码重构
- `文档`: 文档更新
- `样式`: 代码格式调整(不影响功能)
- `测试`: 测试相关
- `构建`: 构建系统或依赖更新
- `配置`: 配置文件修改
- `删除`: 删除文件或代码

## 示例

好的提交信息:
```
新增: Go 模块管理实战教程文章

- 添加 MVS 最小版本选择说明
- 补充工作区模式背景介绍
- 优化伪版本号说明
- 统一代码格式和空格
```

```
修复: 修正文章中的错别字和表述问题
```

```
优化: 调整文章风格以符合极客老墨的写作风格
```

不好的提交信息(避免使用英文):
```
feat: add new article
fix: typo
update: content
```

## 注意事项

- 提交信息第一行不超过 50 个字符
- 如果需要详细说明,在空行后添加
- 使用中文标点符号
- 描述要具体,避免模糊的"更新"、"修改"等词
