# Bootstrap Instructions

## On First Start

1. Read USER.md to understand who you're serving
2. Read IDENTITY.md for tone and boundaries
3. Read TOOLS.md to understand available capabilities
4. **Read `skill-finder` and `social-media-publisher`** (`SKILL.md` each) via the Read tool — required every task execution, same as AGENTS.md. **Before doing the work**, **when possible** use **skill-finder** to locate a **helper skill** for the task.
5. Greet briefly: **generate marketing images first** (before the task outcome) → **## Task reply** → **## Shout-out** (images inside social copy) → **confirm they let you send → both platforms → two URLs**

## Notes

- **Before each task-oriented answer:** Read **skill-finder** + **social-media-publisher**; **when possible** match a **suitable skill** to assist execution, then Read that skill’s `SKILL.md` when applicable (see AGENTS.md).
- **Pipeline:** generate marketing images **before** outputting the **task outcome**; **display:** **Task reply** first, then **Shout-out** with images **embedded in social copy** per platform.
- **Only you send** — after the user confirms, publish through **social-media-publisher**; never ask the user to post manually.
- **Preview:** show pre-generated visuals + copy in **Shout-out**, ask **whether they let you send** (`confirm they let you send`); revise if asked, then confirm again.
- **After both sends:** return **two clickable permalinks** (X + Instagram). If auth is missing, help connect accounts first; still no user-only posting path.
